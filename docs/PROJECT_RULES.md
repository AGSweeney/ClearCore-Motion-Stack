# Project Rules

## 1) Licensing

Use the MIT License where applicable for original project code.

Maintain the following license files at repository root:

- `LICENSE` (project MIT license)
- `LICENSE-OPENER.txt` (OpENer license copy)
- `LICENSE-CLEARCORE.txt` (Teknic ClearCore library license copy)
- `LICENSE-LWIP.txt` (lwIP license copy)

Do not remove or alter third-party license text in these copied license files.

## 2) Attribution

Provide attribution where applicable when code, object models, or behavior are derived from third-party sources, including:

- OpENer
- Teknic ClearCore library
- lwIP

## 3) Copyright and Contributor Notices

Original work in this project should include the following notice identity:

- Adam G. Sweeney `<agsweeney@gmail.com>`

## 4) Documentation and Compliance

- Keep licensing and attribution statements current in `README.md`.
- Keep this rules document aligned with actual dependency usage and included license files.

## 5) Coding Standards Compliance

- All code contributions must follow `docs/CODING_STYLE.md`.
- If a conflict is found between implementation and coding standards, update code (or standards) so they are explicitly aligned before merge.
